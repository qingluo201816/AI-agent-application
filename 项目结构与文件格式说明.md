# AI-agent-application 项目结构与文件格式说明

本文档用于帮助你快速理解仓库的整体结构，以及仓库中各类文件格式的作用与使用场景。

## 1. 项目整体结构

```text
AI-agent-application/
├─ src/                              # 主后端（Spring Boot + Spring AI）
│  ├─ main/
│  │  ├─ java/com/yupi/yuaiagent/    # 业务代码：控制器、Agent、RAG、Tools、配置等
│  │  └─ resources/                  # 配置文件、MCP 配置、RAG 文档语料
│  └─ test/java/com/yupi/yuaiagent/  # 单元测试 / 集成测试
├─ yu-ai-agent-frontend/             # 前端（Vue 3 + Vite）
│  ├─ src/                           # 页面、组件、路由、API 封装
│  ├─ public/                        # 静态资源
│  └─ Dockerfile / nginx.conf        # 前端容器化与部署配置
├─ yu-image-search-mcp-server/       # 独立 MCP 服务（图片搜索）
│  ├─ src/main/java/                 # MCP 服务端代码
│  ├─ src/main/resources/            # 多运行模式配置（stdio / sse）
│  └─ src/test/java/                 # MCP 服务测试
├─ pom.xml                           # 主 Maven 聚合/后端构建配置
├─ Dockerfile                        # 主服务容器构建文件
├─ README.md                         # 项目说明文档
└─ mvnw / mvnw.cmd                   # Maven Wrapper 启动脚本
```

---

## 2. 各目录职责说明

### 2.1 根目录（后端主工程）
- **`src/main/java`**：Java 业务代码核心目录。
- **`src/main/resources`**：运行配置、MCP 服务注册文件、RAG 文档数据。
- **`src/test/java`**：后端测试代码。
- **`pom.xml`**：依赖管理、插件配置、构建生命周期。
- **`Dockerfile`**：后端镜像构建流程。

### 2.2 `yu-ai-agent-frontend`（前端工程）
- **`src/views`**：页面级组件（如 Home、SuperAgent 等）。
- **`src/components`**：可复用组件（如聊天房间、页脚等）。
- **`src/router`**：路由映射。
- **`src/api`**：前端请求封装。
- **`vite.config.js`**：Vite 构建与开发服务器配置。
- **`Dockerfile` + `nginx.conf`**：前端静态构建与 Nginx 托管配置。

### 2.3 `yu-image-search-mcp-server`（MCP 子服务）
- 提供独立的图片搜索工具服务，供主智能体通过 MCP 协议调用。
- 支持不同运行模式（如 stdio、SSE）配置。

## 3. 各文件格式的具体作用

> 下表按“仓库中实际出现的文件格式”进行说明。

| 文件格式 | 典型文件 | 具体作用 |
|---|---|---|
| `.java` | `src/main/java/...` | 后端核心业务代码：Controller、Agent 推理流程、RAG 检索增强、工具调用、配置与启动类。 |
| `.vue` | `yu-ai-agent-frontend/src/views/*.vue` | Vue 单文件组件，整合模板（HTML）、脚本（JS）和样式（CSS），用于页面和组件开发。 |
| `.js` | `yu-ai-agent-frontend/src/main.js` | 前端入口、路由/接口初始化、构建配置脚本（如 `vite.config.js`）。 |
| `.yml` | `src/main/resources/application.yml` | Spring Boot / MCP 服务配置（端口、模型、数据源、运行模式等）。 |
| `.json` | `src/main/resources/mcp-servers.json` | 结构化配置数据，常用于服务注册、前后端依赖元信息和锁定依赖。 |
| `.xml` | `pom.xml` | Maven 构建描述文件，定义依赖、插件、构建步骤和打包方式。 |
| `.md` | `README.md` | 项目说明文档：功能介绍、技术栈、使用方式与学习资料。 |
| `.css` | `yu-ai-agent-frontend/src/style.css` | 前端全局或模块化样式。 |
| `.html` | `yu-ai-agent-frontend/index.html` | 前端宿主 HTML 模板，Vite 在构建时注入打包产物。 |
| `.svg` | `public/vite.svg` | 矢量图资源（Logo / 图标），适合无损缩放。 |
| `.conf` | `yu-ai-agent-frontend/nginx.conf` | Nginx 服务配置（静态资源托管、路由回退、缓存策略等）。 |
| `.properties` | `.mvn/wrapper/maven-wrapper.properties` | Java/Maven 生态中的键值配置文件，用于 Maven Wrapper 版本与下载地址。 |
| `.cmd` | `mvnw.cmd` | Windows 下的启动脚本（例如执行 Maven Wrapper）。 |
| 无扩展名脚本 | `mvnw` | Unix/Linux/macOS 下的可执行脚本，用于统一 Maven 版本执行。 |
| 点文件（无扩展） | `.gitignore`、`.gitattributes`、`.dockerignore` | 工程管理文件：控制 Git 忽略规则、Git 属性、Docker 构建上下文过滤。 |
| 锁文件 | `package-lock.json` | 锁定 Node 依赖精确版本，保证团队环境安装一致性和可复现构建。 |

## 4. 与 AI 功能强相关的关键文件类型（建议优先关注）

1. **Java 业务代码（`.java`）**
   - 智能体执行逻辑（Agent）
   - Tool Calling 工具实现
   - RAG 检索增强链路
2. **Spring 配置（`.yml`）**
   - 模型接入参数、服务端口、环境切换
3. **MCP 配置（`.json` / `.yml`）**
   - MCP 服务注册、通信方式和工具暴露配置
4. **前端页面组件（`.vue`）**
   - 聊天交互体验、Agent 页面展示、接口联调入口

## 5. 新同学阅读顺序建议

1. 先看 `README.md` 了解项目目标与技术栈。
2. 再看根目录 `pom.xml` + `src/main/resources/application.yml` 明确后端依赖与配置。
3. 阅读 `src/main/java` 下的 Controller → Agent → Tools → RAG 包，理解核心调用链。
4. 查看 `yu-ai-agent-frontend/src/views` 和 `src/components`，串联前端交互流程。
5. 最后看 `yu-image-search-mcp-server`，理解外部能力如何通过 MCP 挂载给主智能体。
